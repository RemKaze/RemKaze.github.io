特征工程

“More data beats clever algorithms, but better data beats more data.”
           –Peter Norvig

  • 数据科学最具创意的方面
  • 你应该像对待其它创造性工作，像是写喜剧一样对待它
  • 头脑风暴
  • 创造特征和公示
  • 反复确认起是什么起了作用

分类型特征

  • 总是需要处理
  • 高基数(维度)导致特征稀疏
  • 很难处理缺失值

独热编码(Onehot encoding)

  • One-of-K encoding on an array of length K.
  • 基本用法:与大部分线性算法一起使用
  • 离散后避免共线问题
  • 稀疏格式对内存友好

哈希编码(Hash encoding)

  • 避免数据过于稀疏
  • 可能产生碰撞
  • 可以多次使用不同的哈希函数增加精度
  • 碰撞通常会影响结果，时好时坏
  • 可以优雅的处理新变量

标签编码(Label encoding)

  • 为每个分类变量赋予唯一的数字ID
  • 一般用于基于分类树的非线性算法
  • 不会增加维度
  • 可以随机映射，反复训练取平均值来减小精度上的波动

频数编码(Count encoding)

  • 用统计值来代替属性值
  • 适用于线性或非线性算法
  • 对离群点敏感
  • 对统计值进行对数变换效果也很好
  • 用1填充空值
  • 在统计值相同时会产生碰撞

标签频数编码(LabelCount encoding)

  • 根据在训练集中的属性的统计值排名
  • 适用于线性或非线性算法
  • 对离群点不敏感
  • 不产生碰撞(看例子)
  • 两全其美

目标编码(Target encoding)

  • 用属性在目标上的比率进行编码(二分或回归)
  • 警惕过拟合
  • 进行交叉验证
  • 增添平滑避免将特定类别编码为0
  • 添加噪声避免过拟合
  • 正确操作时适用于适用于线性或非线性算法

嵌入(Category Embedding)

神经网络部分，不懂

  • Use a Neural Network to create dense embeddings
    from categorical variables.
  • Map categorical variables in a function approximation
    problem into Euclidean spaces
  • Faster model training.
  • Less memory overhead.
  • Can give better accuracy than 1-hot encoded.
  • https://arxiv.org/abs/1604.06737

空值处理(NaN encoding)

  • 为空值提供明确的编码而不是无视
  • 空值也含有信息
  • 警惕过拟合
  • 仅当训练集和测试集中的NaN值是由相同的原因引起时，或当本地验证证明它含有信息时使用

多项式编码(Polynomial encoding)

  • 对变量间的关系编码(见例子)
  • 不关注变量间关系的线性分类器无法解决XOR问题
  • 一个多项式的核“能”解决XOR问题
  • 扩展特征空间

扩展编码(Expansion encoding)

  • 从单属性中分解出更多特征
  • 部分特征含有大量次级特征(见例子)

合并编码(Consolidation encoding)

  • 将不同种类的变量映射到同一个变量
  • 像是拼写错误，略微不同的工作描述，全名和缩写
  • 真实的文本很混乱，尤其是自由(格式)文本
